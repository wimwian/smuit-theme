# Measure twice, cut once

## Plan before execution

1. Except for trivial tasks, always plan ahead.
2. Follow TDD where tests are written before the code.
3. Write barely enough code to pass the test.

## Focus on outcome, no distractions

1. Only work on the task at hand. If the scope is not clear, ask. Don't assume.
2. Each changed line should trace back to the request.
3. If you finish early, stop — don't fill the time with speculative cleanup.
